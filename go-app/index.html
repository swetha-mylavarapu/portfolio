Go app differs from other shared driving services with complete isolation of data and details about users. The framework is built by compliance with identity access management principles in focus.
Go App provides users with a variety of options for choosing their vehicles and options of tagging additional riders for their journeys. The application makes travel of the users within a locality more convenient and personalized.
In order to evaluate the application features, I had ventured out to study the possible applications present currently and the features they provide making the travel as convenient as possible for the users and analyzed the gap so that I can make Go application enhance the user travel experience more convenient. I evaluated several features that seemed critical from user surveys and identified which features in the application could be capitalized on to have an edge over other applications.

I found that primarily users rely on Shared riding options  for travel like Grab/ Uber where the users can rent/hire a vehicle for their travel from one place to another.  The other options are renting bicycles, e-scooters, etc where the users can unlock a bike of their choice from the vendor specified location and drop-off the bike at the destination spot. Even with these options currently available, majority of users felt that during the rides if they want to have private rides and make ad-hoc stops during the travel and such options where not available.I place Go application here for such users as the Go app is designed with lot of privacy features and personalized travel experience
Go is a mobile application providing shared locomotive services for users to travel from one location to another.
The application provides a framework for users to share/rent  bikes for their use.
